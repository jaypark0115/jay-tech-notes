SoC/FPGA Bus 학습 노트

이 노트는 숭실대학교 차세대반도체학과 동계 학부생연구인턴에서 진행한 APB, AHB, AXI, AXI-to-APB Bridge 실습을 정리한 글입니다. 단순히 bus 이름을 설명하는 페이지가 아니라, 수업자료의 register map과 Lab 코드, SimVision 파형, 최종 프로젝트 로그를 기준으로 내가 어떤 순서로 이해했고 어떤 부분을 직접 확인했는지 남겼습니다.

읽는 순서 APB에서 register access를 먼저 잡고, AHB에서 phase와 wait-state를 본 뒤, AXI의 channel handshake를 이해하고, 마지막으로 AXI-to-APB Bridge에서 protocol 변환을 구현한 흐름입니다.
APB APB Control Bus register map, setup/enable phase, SRAM, interrupt 실습 AHB AHB Pipelined Bus master/decoder/slave, address/data phase, SP-SRAM wait-state AXI AXI Channel Bus AW/W/B, AR/R, VALID/READY handshake, Lab7 AXI slave BRG AXI-to-APB Bridge write/read FSM, APB slave 4-register target, testbench PASS 검증

수업자료 기준 Lab 진행 흐름

전체 과정은 APB register 실습에서 시작해 SRAM과 interrupt로 확장되고, AHB의 phase 구조, AXI의 channel 구조, 최종 AXI-to-APB protocol conversion으로 이어졌다. 이 순서가 중요하다. APB를 먼저 해 봐야 register map과 setup/enable을 알 수 있고, AHB를 거치며 phase와 wait-state를 이해한 뒤, AXI에서 channel별 handshake를 분리해서 볼 수 있었다. 최종 프로젝트는 이 세 가지를 한 번에 묶어 AXI 요청을 APB register 접근으로 바꾼 작업이었다.

Lab2~4

APB 기반 register/SRAM/interrupt

주소 decode, write commit, read mux, start command, pending/mask register를 실습했다.

Lab5~6

AHB register와 SP-SRAM

address/data phase, HREADY, zero-wait write와 one-wait read를 확인했다.

Lab7

AXI slave register block

AW/W/B와 AR/R 채널을 분리해 VALID/READY handshake를 검증했다.

Project

AXI-to-APB Bridge

write/read FSM으로 protocol을 변환하고 4-register readback PASS를 확인했다.
